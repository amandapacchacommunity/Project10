Documentation

How this analysis is built

This project combines direct public datasets with reference documents and contextual materials. Because public sources differ in format and level of detail, the method emphasizes documentation, transparent assumptions, and cautious interpretation.

Core Workflow

  1. Collect public workforce, vacancy, classification, governance, and budget materials.
  2. Clean and normalize department names and title fields where needed.
  3. Create standardized role groups using documented crosswalk logic.
  4. Apply derived tags for governance relevance and 2022 structural signals.
  5. Summarize results using descriptive metrics and an illustrative risk lens.

Data Use Standard

  • Direct source fields are reported as published.
  • Normalized titles and role groups are derived through documented mapping logic.
  • Governance and structural indicators are used as analytical signals, not official determinations.
  • Where public information is incomplete, limitations are stated explicitly rather than filled with unsupported assumptions.

Confidence Framework

  • High confidence: directly supported by published dataset fields or official reference materials.
  • Moderate confidence: supported by crosswalk logic or multiple public sources.
  • Exploratory: inferred from budget language, structural context, or broader public documentation.

Illustrative Risk Framework

Illustrative Risk Index = Classification Consistency Signal + Vacancy Pressure Signal + Compensation Comparison Signal + Structural Change Signal

This is a portfolio model designed to organize public administrative signals into a consistent analytical framework. It is not an official institutional risk rating.

Assumptions

  • Some title mappings require manual standardization.
  • Not all governance or structural materials are row-level datasets.
  • Some indicators are best understood as contextual signals rather than precise measurements.

Limitation

Public data and reference documents may contain ambiguity, formatting differences, or incomplete role detail. This project prioritizes transparency over false precision.
