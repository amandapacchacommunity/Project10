Documentation

Source log format

Each source should be documented with its publisher, type, date accessed, and exact use in the project. This helps distinguish direct dataset fields from context documents and manually coded indicators.

Documentation Standard

  • Record the exact title of each source.
  • Note whether the source is a machine-readable dataset, PDF, webpage, or memo.
  • Document the specific variable, tag, or interpretation it supports.
  • Keep a note on any manual extraction or inferred logic used.

Transparency Note

The strength of this project comes partly from clear documentation. Even when some evidence comes from PDFs or memos rather than structured tables, careful source logging makes the analysis more credible and easier to defend in an interview or portfolio review.
